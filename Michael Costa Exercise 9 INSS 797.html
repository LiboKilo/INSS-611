A Support Vector Machine (SVM) is a powerful Machine Learning model, capable of performing linear or nonlinear classification, and regression. It is one of the most popular models in Machine Learning, and any one interested in Machine Learning should have it in their toolbox. In this document, we will learn how to implement SVM in Python.

Linear SVM Classification¶

Linear SVM models are applicable when two classes are clearly linearly separable. The implementation of SVM is similar to the processes of implementing other ML models we learned before. The following cell loads the iris dataset (introduced previously), scales the features (neccessary for SVM), and then trains a linear SVM model to detect Iris virginica flowers:

In [1]:
import warnings
warnings.simplefilter(action='ignore')
import numpy as np
from sklearn import datasets
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import LinearSVC
from sklearn.model_selection import train_test_split

iris = datasets.load_iris()
X = iris["data"][:, (2, 3)] # petal length, petal width
y = (iris["target"] == 2).astype(np.float64) # Iris virginica
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
svm_clf = Pipeline([
("scaler", StandardScaler()),
("linear_svc", LinearSVC(C=1, loss="hinge")),
])
svm_clf.fit(X_train, y_train)

                ('linear_svc', LinearSVC(C=1, loss='hinge'))])

Nonlinear SVM Classification¶

Although SVM can normally perform well on linearly separable data, it can also handle nonlinear data excellently using different kernels. Let's start with adding extra polynomial features in the model. Run the following cell to create and plot a double-moon data:

In [4]:
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons
X, y = make_moons(n_samples=100, noise=0.15, random_state=42)
def plot_dataset(X, y, axes):
    plt.plot(X[:, 0][y==0], X[:, 1][y==0], "bs")
    plt.plot(X[:, 0][y==1], X[:, 1][y==1], "g^")
    plt.axis(axes)
    plt.grid(True, which='both')
    plt.xlabel(r"$x_1$", fontsize=20)
    plt.ylabel(r"$x_2$", fontsize=20, rotation=0)
plot_dataset(X, y, [-1.5, 2.5, -1, 1.5])
plt.show()

The following cell creates third-degree polynomial features and integrates this process into a pipeline for a SVC on the double-moon data we just created:

def plot_predictions(clf, axes):
    x0s = np.linspace(axes[0], axes[1], 100)
    x1s = np.linspace(axes[2], axes[3], 100)
    x0, x1 = np.meshgrid(x0s, x1s)
    X = np.c_[x0.ravel(), x1.ravel()]
    y_pred = clf.predict(X).reshape(x0.shape)
    y_decision = clf.decision_function(X).reshape(x0.shape)
    plt.contourf(x0, x1, y_pred, cmap=plt.cm.brg, alpha=0.2)
    plt.contourf(x0, x1, y_decision, cmap=plt.cm.brg, alpha=0.1)

plot_predictions(polynomial_svm_clf, [-1.5, 2.5, -1, 1.5])

Polynomial Kernel¶

As we learned from the slides, for SVM, we can implement the polynomial transformation without really adding extra polynomial features in model using so-call kernel trick. The kernel for polynomial transformation is called the polynomial kernel. This trick is implemented by the SVC class. Lets test it on the moons dataset:

In [7]:
from sklearn.svm import SVC
poly_kernel_svm_clf = Pipeline([
        ("scaler", StandardScaler()),
    #Set the kernel to be poly and degree to 3.
        ("svm_clf", SVC(kernel="poly", degree=3, coef0=1, C=5))
    ])
poly_kernel_svm_clf.fit(X, y)
svc_poly_scores = cross_val_score(poly_kernel_svm_clf, X, y,
                            scoring="accuracy", cv=10)
print("The average accuracy score is:",svc_poly_scores.mean())

The average accuracy score is: 0.96

previously we got 95%, in this above cell, polynomial trick increased the accuracy score by 4%¶

As you learned from the slides, you can fine tune the following hyperparameters for a poly kernel SVC:

  1. degree (degree of the polynomial transformation)
  2. coef0 (how much the model is influenced by high-degree polynomials)
  3. C (penalty for misclassifications)

Reduce degree and C will reduce the overfitting problems. Let's conduct the hyparameter tuning for the SVC above using the random search approach:

In [8]:
poly_kernel_svm_clf = SVC(kernel="poly")
rs_hypers={ 'degree':list(np.arange(1, 10)),
             'coef0':list(np.arange(1, 100,step=5)),
              'C':list(np.arange(1, 20))}
# Run random search
from sklearn.model_selection import RandomizedSearchCV
random = RandomizedSearchCV(poly_kernel_svm_clf, rs_hypers, n_iter=300, scoring='accuracy', cv=3)
standard_scaler = StandardScaler()
X_standardized = standard_scaler.fit_transform(X)
model_random = random.fit(X_standardized,y)

# Print out the results
print('Best hyperparameters are: '+str(model_random.best_params_))
print('Best score is: '+str(model_random.best_score_))

Best hyperparameters are: {'degree': 3, 'coef0': 11, 'C': 9}
Best score is: 0.9702911467617351

Similarity Features: Gaussian RBF Kernel¶

Another technique to tackle nonlinear problems is to add extra features computed using a similarity function. Just like the polynomial features, kernel trick does its SVM magic, making it possible to obtain a similar result as if you had added many similarity features. Lets try the SVC class with the Gaussian RBF kernel:

In [9]:
rbf_kernel_svm_clf = Pipeline([
("scaler", StandardScaler()),
("svm_clf", SVC(kernel="rbf", gamma=5, C=0.001))
])
rbf_kernel_svm_clf.fit(X, y)
svc_rbf_scores = cross_val_score(rbf_kernel_svm_clf, X, y,
                            scoring="accuracy", cv=10)
print("The average accuracy score is:",svc_rbf_scores.mean())

The average accuracy score is: 0.9700000000000001

The hyperparameters for RBF kernel are gamma and C. Increasing gamma makes the bell-shaped curve narrower. If your model is overfitting, you should reduce it; if it is underfitting, you should increase it (similar to the C hyperparameter). Let's perform a grid search for a RBF kernel SVC:

In [10]:
# Grid search hyperparameters:
grid_hypers ={'gamma':[0.01, 0.1, 1, 5, 10],
              'C':[0.001, 1, 100, 1000, 5000]}

# Run Grid Search
from sklearn.model_selection import GridSearchCV
rbf_kernel_svm_clf = SVC(kernel="rbf")
grid = GridSearchCV(rbf_kernel_svm_clf,param_grid=grid_hypers,cv=3,scoring='accuracy')
model_grid = grid.fit(X_standardized,y)

# Print out the result
print('Best hyperparameters are: '+str(model_grid.best_params_))
print('Best score is: '+str(model_grid.best_score_))

Best hyperparameters are: {'C': 1000, 'gamma': 0.1}
Best score is: 0.9702911467617351

SVM Regression¶

Like the decision tree algorithm, SVM supports both classification and regression. To use the SVMs for regression, the trick is to reverse the objective: instead of trying to fit the largest possible street between two classes while limiting margin violations, SVM Regression tries to fit as many instances as possible on the street while limiting margin violations (i.e., instances off the street). The width of the street is controlled by a hyperparameter, (epsilon).
