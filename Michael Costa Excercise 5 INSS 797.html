In this week, let's explore how to implement time series forecasting analyses in Python. As the slides covered, there are two types of forecasting models: moving average-based and regression-based models.

Moving Average¶

Let's start with moving average methods since they are more straightforward. As we learned, moving average models make predictions by averaging past values. Therefore, there is no need to "train" moving average models.

First, let's talk about the simple moving average, which is calculated by taking the arithmetic mean of a given set of values over a specified period. For example, the following cell predicts the stock price for Apple company by averaging its past 250 days close prices using the pandas' rolling() function. The function takes one argument for the size of the moving windows. In this section, we use the stock market price dataset. It contains the close prices for four stocks from 2003 to 2011.

Moving windows' sizes determine the level of generality. The larger a wondow size, the more general the patterns will be. It is reasonable to assume that you can observe more details from a moving average for the past 50 days than it is for the past 200 days. It is the analyst's judgement to decide the size of a moving window. People tend to select smaller window, when they want to show short-term changes, and vice versa.

The following cell displays plots for two moving averages with the sizes of 50 and 250 days for the Apple stock prices. Can you see the differences?

Exponential Moving Average¶

Unlike the simple moving average giving all past observations equal weight when averaging, exponential moving average specify a constant decay factor to give more weigt to more recent observations. Since an exponential moving average places more weight on more recent observations, it adapts faster to changes compared with the simple moving average.

pandas has the "ewm()" function for that. It defines the moving window's size by passing the span= argument. The following cell displays the moving average lines using the two different methods with a window size equals 30 for the Apple price between 2006 and 2007.

In [4]:
import matplotlib.pyplot as plt
aapl_px = close_px.AAPL['2006':'2007']
sma60 = aapl_px.rolling(30, min_periods=20).mean()
ema60 = aapl_px.ewm(span=30).mean()
sma60.plot(style='b-', label='Simple Moving Average')
ema60.plot(style='r--', label='Exponential Moving Average')
plt.legend()

Time Series Forecasting using sklearn¶

In the first half of the document, we learned how to implement simple and exponential moving average models using pandas in Python. They are straightforward since their machanisms are relatively simple. We use the (weighted) average to as predictions for the future. Therefore, there is no need to train any models.

On the other hand, the other set of forecasting models are regression-based. As we learned from the slides, regresssion-based models use the past values (aka. lags) as predictors and fit them into a regression models to predict the future values. For those models, it is neccessary to train the models with some data. This pattern is closer to the concept of machine learning. Thus, in the second half of this document, let's learn how to use sklearn to train regression models for time series data.

First, let's look at the data. The dataset we use is for the Germany electricity consumption from 2006 to 2017. Run the following cell to read the data.

Cross Validation for Time Series Models¶

One special point of time series models is that we cannot use the traditional k-fold cross validation to evaluate models since we must follow a time order.

Instead, we can perform a Forward Chaining strategy. For example, the training data is from 2018 to 2021. Then, we seperate the training data into four years, the train and validation sets look like:

  1. fold 1: training [2018], validation [2019]
  2. fold 2: training [2018,2019], validation [2020]
  3. fold 3: training [2018, 2019, 2020], validation [2021]

This way enables us to have mutliple iterations for validation with a correct time order. sklearn provides such a train-test split using TimeSeriesSplit. The TimeSerieSplit function takes the number of splits as input. Since our training data has 11 unique years (2006 -2016), we would be setting n_splits = 10. This way we have 10 training and validation sets.

In [9]:
from sklearn.model_selection import TimeSeriesSplit
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import cross_val_score
import numpy as np
#Let's start with linear regression model:
lrts = LinearRegression()
# TimeSeries Cross validation
tscv = TimeSeriesSplit(n_splits=10)
lr_cv_r2 = cross_val_score(lrts, X_train, y_train, cv=tscv, scoring='r2')
lr_cv_nmse = cross_val_score(lrts, X_train, y_train, cv=tscv, scoring='neg_mean_squared_error')
print("R-squared: ", lr_cv_r2.mean())
print("RMSE: ", np.sqrt(-lr_cv_nmse).mean())

import matplotlib.pyplot as plt
from sklearn.neural_network import MLPRegressor
from sklearn.neighbors import KNeighborsRegressor
from sklearn.ensemble import RandomForestRegressor
from sklearn.svm import SVR
models = []
models.append(('LR', LinearRegression()))
models.append(('MLP', MLPRegressor(solver = 'lbfgs')))  #neural network
models.append(('KNN', KNeighborsRegressor())) 
models.append(('RF', RandomForestRegressor(n_estimators = 10))) # Ensemble method - collection of many decision trees
models.append(('SVR', SVR(gamma='auto'))) # kernel = linear
# Evaluate each model in turn
results = []
names = []
for name, model in models:
    # TimeSeries Cross validation
    tscv = TimeSeriesSplit(n_splits=10)
    
    cv_results = cross_val_score(model, X_train, y_train, cv=tscv, scoring='r2')
    results.append(cv_results)
    names.append(name)
    print('%s: %f (%f)' % (name, cv_results.mean(), cv_results.std()))
    
# Compare Algorithms
plt.boxplot(results, labels=names)
plt.title('Algorithm Comparison')
plt.show()
