Decision Trees can perform both classification and regression tasks, and even multioutput tasks. They are powerful algorithms, capable of fitting complex datasets.

In this document we will start by discussing how to train, visualize, and make predictions with Decision Trees. Then we will go through the CART training algorithm used by Scikit-Learn, and we will discuss how to regularize trees and use them for regression tasks.

Training and Visualizing a Decision Tree¶

Let's first build a decision tree and take a look at how it makes predictions. The following cell trains a DecisionTreeClassifier on the iris dataset. The iris dataset has four attributes: sepal length, sepal width, petal length, and petal width. The three types of iris are: setosa, veriscolor, and virginica. Let's try to use the four attributes to identify the species for the iris.

In [4]:
# Load the built-in iris data.
from sklearn.model_selection import train_test_split
from sklearn.datasets import load_iris
from sklearn.tree import DecisionTreeClassifier
# Store the dataset as "iris".
iris = load_iris()
# Predictors
X = iris.data
# Targets
y = iris.target
# Randomly sample our data --> 70% to train with, and 30% for testing
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3)
# Build a decision tree model and set the maximum depth to be two.
tree_clf = DecisionTreeClassifier(max_depth=2)
# Train the model
tree_clf.fit(X_train, y_train)

Feature Importance¶

As we learned, the features that appears at the higher levels in a deicion trees are more important. Actually, we can directly show the feature importance using the featureimportances attributes on a decision tree model. The importance is a number between 0 and 1 for each feature, where 0 means not used at all and 1 means perfectly predicts the target. The feature importances always sum to 1:

In [7]:
print("Feature importances:",tree_clf.feature_importances_ ,sep='\n')

Feature importances:
[0. 0. 0. 1.]

As you can tell, the first two features (sepal length and width) are not used at all. Meanwhile, the last two features (petal length and width) have similar impacts on the target. In other words, petal attributes are most important to identify different iris species. We can visualize the feature importances:

In [8]:
import numpy as np
def plot_feature_importances(model):
    n_features = iris.data.shape[1]
    plt.barh(np.arange(n_features), model.feature_importances_, align='center')
    plt.yticks(np.arange(n_features), fn)
    plt.xlabel("Feature importance")
    plt.ylabel("Feature")
    plt.ylim(-1, n_features)

Making Predictions¶

Lets see how the tree represented in figure above makes predictions.At each split, the left branch means True to the condition and the right one is for False. Suppose you find an iris flower and you want to classify it. You start at the root node (depth 0, at the top): this node asks whether the flowers petal width is smaller than 1.75 cm. If it is not, then you move down to the roots right child node (depth 1, right). In this case, it is a leaf node node (i.e., it does not have any child nodes), so it does not ask any questions: simply look at the predicted class for that node, and the Decision Tree predicts that your flower is an Iris virginica (class=virginica).

Now suppose you find another flower, and this time the petal width is less than 1.75 cm. You must move down to the roots left child node (depth 1, left), which is not a leaf node, so the node asks another question: is the petal length smaller than 2.45 cm? If it is not, we move further down to its right and reach the leaf node. Based on the information in the leaf node, it is most likely an Iris versicolor (depth 2, right). Its really that simple. Unlike regression models predictions are numbers, classification models predictions are classes. Let's make predictions on the test data:

Estimating Class Probabilities¶

Although the predicted values are classes, decision tree models predict based on the class probabilities. For example, if the decision tree output the following probabilities: 0% for Iris setosa, 92.5% for Iris versicolor, and 0.3% for Iris virginica, it will predict the instance as Iris versicolor (class 1) because it has the highest probability.

Actually, we can check the probabilities for different classes using predict_proba function:

In [11]:
print("Class probabilities:",tree_clf.predict_proba([[5, 2.5, 2.5, 0.3]]))
print("Predicted class:", tree_clf.predict([[5, 2.5, 2.5, 0.3]]))

Tree Pruning: Hyperparameters Tuning¶

To avoid overfitting the training data, we need to restrict the Decision Trees freedom during training. The DecisionTreeClassifier class has several parameters that similarly restrict the shape of the Decision Tree:

  1. max_depth (maximum depth of the decision tree).
  2. min_samples_split (the minimum number of samples a node must have before it can be split).
  3. min_samples_leaf (the minimum number of samples a leaf node must have).
  4. min_weight_fraction_leaf (same as min_samples_leaf but expressed as a fraction of the total number of weighted instances).
  5. max_leaf_nodes (the maximum number of leaf nodes)
  6. max_features (the maximum number of features that are evaluated for splitting at each node).
  7. criterion (either "gini" (default) or "entropy")

In general,increasing min_ hyperparameters or reducing max_ hyperparameters will restrict the growth of the model, aka prune the tree. We can use grid search or random search to find the optimal hyperparameter combinations, as we learned briefly in the first-look document. The following is an example for the wine quality dataset.
